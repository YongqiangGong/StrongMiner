## 📖 引言 

StrongMiner 是一款面向习题集内容结构化提取的专用框架，依托大语言模型（LLMs）的强大语义理解能力，可自动解析 PDF 习题集的目录层级与章节划分，精准提取题目与配套答案，并将其转化为标准化 JSON 结构化数据，为后续学术研究、题库建设等场景提供高质量的数据支撑。

该框架创新性地融入人机交互流程，不仅有效弥补了纯自动化提取的精度短板，更针对性地解决了直接调用大模型时面临的上下文窗口限制问题—— 这一问题曾导致大模型无法一次性输出完整的结构化数据，极大影响了长文档、厚习题集的处理效率与完整性。通过多步骤的人机协同处理，StrongMiner 大幅提升了习题集内容的提取效率与准确性。

目前，StrongMiner 已在医学习题集的实际应用中完成充分验证，凭借稳定的性能与高精度的输出效果，展现出良好的实用性与可扩展性。

## ✨ 功能特点

- **📖 目录识别**：自动识别文档中的目录页面，区分目录页与正文内容  
- **챕 章节提取**：从目录页中提取章节名称和对应的页码范围  
- **❓ 题目识别**：智能判断页面内容是否包含题目或答案  
- **✅ 答案提取**：根据题目识别结果，提取对应的答案内容  
- **🏷️ 标签注释**：根据题目内容，注释所属的对应标签  
- **🧾 结构化输出**：将提取的信息转换为标准化的 JSON 格式

## 安装与使用

### 1. 创建新环境并激活

```bash
conda create -n StrongMiner python=3.12
conda activate StrongMiner
```

### 2. 安装依赖

padleocr安装请参考：https://github.com/PaddlePaddle/PaddleOCR

其余依赖：

```bash
pip install instructor openai pydantic langgraph
# 速度较慢可使用镜像，推荐阿里云镜像
```

### 3. 下载项目至工作路径

```bash
git clone https://github.com/YongqiangGong/StrongMiner.git
cd StrongMiner
```

### 4.开始运行

```bash
python langgraph_code.py
# 或运行 my_code.py 文件
```

### 5. 工作流程及人机交互（以医学习题集为例）
StrongMiner 针对习题集的结构化提取设计了分阶段执行流程，全程融入人机交互校验环节保障数据准确性，以下为医学习题集场景下的完整操作流程：

#### 🔧 5.1 初始配置输入
运行框架主程序后，需依次手动输入核心配置信息（支持绝对路径/相对路径、标准 API 格式）：
- 📄 PDF 源文件路径（示例：/home/user/pdf）
- 📂 结构化数据输出目录（示例：/home/user/output）
- 🌐 大模型服务 API 地址（示例：https://dashscope.aliyuncs.com/compatible-mode/v1）
- 🤖 调用的模型名称（示例：qwen3-max）
- 🔑 模型访问密钥（示例：sk-2d7c12272772）

#### 🔄 5.2 PDF 转 Markdown 预处理
配置完成后，框架自动将 PDF 文件转换为 Markdown 格式：
- 该步骤因文档解析与 OCR 处理耗时较长；
- 若需提升转换效率，可参考 PaddleOCR 官方教程调整代码中 OCR 相关参数。

#### 📑 5.3 目录识别与人工校正
1. 转换完成后，需设定「目录结束最大页码」（默认值为 20，**注意：StrongMiner 页码计数从 0 开始**）；
2. 框架自动扫描识别目录所在页码范围，随后需人工校验并修正「目录起始页码」「目录结束页码」，确认后进入下一阶段。

#### 📘 5.4 章节结构解析与校验
1. 输入正文起始页码（默认值为目录结束页的下一页），框架自动识别各章节名称及对应的页码区间；
2. 章节识别完成后，支持人工检查并修正章节页码范围的准确性，确保后续题目提取范围无偏差。

#### ❓ 5.5 题目定位与人工校正
1. 框架自动解析各章节内题目对应的页码分布；
2. 识别完成后需人工校验题目页码准确性，校正无误后启动「题目提取 + 答案匹配」核心流程。

#### 🏷️ 5.6 题目标签自定义配置
- 框架默认对提取的题目添加多维度结构化标签：题目类型、题干内容、所属人体系统、对应疾病分类；
- 若需适配自定义标签体系（如题型难度、考察知识点等），可修改代码中的提示词模板及标签解析逻辑。

### 6. 输出结果

运行完成后，StrongMiner 将生成以下文件：

- `md`：包含pdf文档对应的markdown格式
- `qa.json`：包含所有题目和答案的结构化数据
- `qatype.json`：包含所有题目所属类型的结构化数据（选择题、填空题、判断题、简答题）
- `qacontent.json`：包含所有题目所属内容的结构化数据（病因病理、诊断治疗、临床表现、文献指南和其他）
- `qasystem.json`：包含所有题目所属人体系统的结构化数据
- `qadisease.json`：包含所有题目所属疾病的结构化数据

这些文件将存储在指定的输出目录中，用户可以根据需要进行后续处理和分析。

## ⚠️ 注意事项

- 🚀 尽量选用超大参数的大模型以提升提取效果，避免使用小参数模型
- 🔗 目前StrongMiner仅支持题目与答案位于同一章节，暂不支持跨章节关联
- ⏱️ 前期pdf文档转换为markdown格式速度较慢，加速请参考padleocr使用教程并自行更改代码（使用Vllm）
- 🔓 本框架为开源项目，用户可以根据需要进行修改和定制。
